03  ·  The one idea to take home

The code output isn't the valuable part anymore. The direction is.

  • With open source, the valuable thing was the code. You shared the code.
  • With AI, what's valuable is how you directed the agent to get there.
  • "Build anything" is real now. The bottleneck moved from typing to steering.

Not a chatbot answer.

Custom plugins Database tables REST APIs Themes WP-CLI commands

Real WordPress, all the way down: no lock-in, no new framework to learn. The same skills you already have (WP-CLI, PHP, git) are now the steering wheel.

05  ·  What the direction looks like

A few sentences of direction.
Thirty minutes of building.

The direction

Take this design and make a custom WordPress theme. Build a custom plugin for the engine — it should have a custom database table.

Those few sentences can run for 30 minutes to an hour. A simple landing page is quick; a custom engine takes a while — but it's not much more direction. Name the architecture, and a few key terms drive some very complicated programming.

Dismissed.FYI  ·  The clever part

Not scraping HTML. Reflecting into the code itself.

01

Admin notices are just callbacks. Enumerate them, then use PHP Reflection to pin the exact file:line.

02

Severity = how much of wp-admin it hijacks. Renders on 3+ unrelated screens = auto-critical.

03

Ranked by intent: Critical = pure vendor benefit, Low = actually serves you.

04

Everything renders in a throwaway local sandbox, fed plugin files from a wp.org mirror. Never a live site.

Dismissed.FYI  ·  Building responsibly

Building fast still means catching your own mistakes.

  • •Early versions rendered notices against live sites.
  • •One nag baked a real customer's account name into a public card.
  • •July 1: hard architecture pivot. Retired all live-site rendering, moved to local sandboxes.
  • •The agent will do exactly what you steer it to. That responsibility stays yours.

18  ·  Tips & tricks for getting the most out of Claude Code
01

Design first. Mock the UI, then let Claude translate it to PHP. Implement against a spec, not a guess.

02

Never touch prod directly. Every project gets a *.localhost site first.

03

Make it verify in a browser. Claude drives Chrome and screenshots both themes before "done."

04

Persist hard-won architecture as memory. Gotchas survive across sessions.

05

Standardize your commits. Emoji-Log: NEW / IMPROVE / FIX / DOC / RELEASE.

06

Self-host updates. A manifest.json self-updater. The GitHub release is the deploy.

07

Mine your own history. command-center indexes 1,400+ past sessions, so your prior AI work becomes a reusable knowledge base.
