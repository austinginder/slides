Distributing Plugin Updates

  • WordPress.org
  • Easy Digital Downloads

### Advantages: on your own - Full control over support - No SVN, development using Git or however you'd like - No limitations on trademarks or coding standards - Hidden by default - No need for readme.txt
### Disadvantages: on your own - Not as trusted to the average user - Lack of discoverablity - No safeguards
### Our your own is easy then you think Here's what you need: - A plugin manifest file `plugin.json` - Two filter hooks `plugins_api` and `site_transient_update_plugins` and some update logic. - Packaged zip of your plugin, uploaded anywhere
No PHP library is required
Fantastic blog post by Misha Rudrastyh: [How to Configure Self-Hosted Updates for Your Private Plugins](https://rudrastyh.com/wordpress/self-hosted-plugin-update.html)
