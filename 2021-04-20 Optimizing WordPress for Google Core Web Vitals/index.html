### Core web vitals deal with how preformat your code is, not your server > "To put it simply, while the amount of code will always matter for performance, you also now have to consider how your website loads the code." [ðŸ”— woorkup.com/speed-up-wordpress/](https://woorkup.com/speed-up-wordpress/)
- **Largest Contentful Paint (LCP)** measures loading performance. To provide a good user experience, LCP should occur within 2.5 seconds of when the page first starts loading. - **First Input Delay (FID)** measures interactivity. To provide a good user experience, pages should have a FID of less than 100 milliseconds. - **Cumulative Layout Shift (CLS)** measures visual stability. To provide a good user experience, pages should maintain a CLS of less than 0.1.
### Where to start - [PageSpeed Insights tool](https://developers.google.com/speed/pagespeed/insights/) - [web.dev Measure tool](https://web.dev/measure/) - [GTmetrix](https://gtmetrix.com/)
### Pagespeed is not web host performance - Fast web hosting will not solve pagespeed issues - Same website, same web host can have vastly different scores with different themes and plugins.
### How to improve? - Existing themes: [WP Rocket](https://wp-rocket.me/), [Jetpack Image CDN](https://jetpack.com/support/site-accelerator/), [How to Speed Up Your WordPress Site (Ultimate 2021 Guide)](https://kinsta.com/learn/speed-up-wordpress/) - Switch to a already preformat theme like [Blocksy](https://creativethemes.com/blocksy/).
